# Опис ER-моделі:

# User (Користувач)

Використовується для зберігання даних облікового запису користувача, які вводяться під час його реєстрації, і з якими система звіряє введені під час авторизації дані.

Має поля:

  • login: TEXT. Містить текст із логіном користувача;

  • email: TEXT. Містить текст із адресою електронної пошти користувача;

  • password: TEXT. Містить текст із паролем користувача.

# Member (Учасник)

Використовується як посередник; визначає те, які права певний користувач має у доступі до певного проєкту.

Немає жодних полів.

Екземпляр учасника відповідає одному проєкту, одному користувачу й одній ролі.

# Role (Роль)

Використовується для визначення ролі учасника. Має поля:

  • slug: TEXT. Містить унікальний рядок ідентифікатор.

Роль може мати від 0 до * прав. Взаємозв'язок ролей і прав визначається у roleGrant

# Grant (Право)

Визначає єдине право використання проєкту.
Має поля:

  • slug: TEXT. Містить унікальний рядок ідентифікатор, зрозумілий людині, з описом права у форматі snake_case.

Початково у базу даних будуть занесені такі права:

  • delete_project
  • add_section
  • delete_section
  • create_task
  • edit_task
  • remove_task
  • accept_task
  • refuse_task
  • add_member
  • change_rights
  • delete_member
  • create_attachment
  • edit_attachment
  • remove_attachment

# RoleGrant

Використовується для призначення прав ролям. А також для вирішення відношення багато-до-багатьох.

# Project (Проєкт)

Використовується для роботи з проєктами. Має поля:

  • name: TEXT. Містить текст з назвою проєкту.
  • description: TEXT. Містить текст з описом проєкту.

Проєкт може мати від 0 до * розділів та від 0 до * учасників. Користувачі можуть мати різні ролі в проєкті, та виконувати різні дії відповідно до них. Коли користувач створює проєкт, йому автоматично надається роль "Замовник", що визначає максимальні права.

# Section (Розділ)

Використовується для зручного сортування завдань. має поля:

  • name: TEXT. Містить текст з назвою розділу.
  • index: NUMBER. Містить номер, що показує на якій позиції на сторінці розміщенна секція.

Розділ може мати від 0 до * завдань.

Особливим і обов'язковим є розділ "Вільні завдання". У ньому знаходяться завдання, що не мають виконавців. Завдання видаляються з цього розділу, коли завдання набуває достатню кількість виконавців.

# Task (Завдання)

використовується для роботи з завданнями.
має поля:

  • name: TEXT. Містить текст з назвою завдання.
  • description: TEXT. Містить текст з описом завдання.
  • deadline: DATE. Містить дату, до якої завдання має бути виконано.
  • index: NUMBER. Містить номер, що показує на якій позиції у Section (Розділі) завдання розміщенно.

Завдання може мати від 0 до * вкладень.
Якщо завдання знаходиться у розділі "Вільні завдання", то відповідно не має виконавця (Executor). Щойно у Task з'являється щонайменше 1 виконавець (Executor), завдання видаляється з розділу "Вільні завдання".
Залежно від ролі користувача у проєкті, користувач може виконувати різні дії з завданнями.

# Attachment (Вкладення)

використовується для вкладення до завдання додаткових файлів різних типів (txt, png, mp4 і тд).
має поля:

  • name: TEXT. Містить текст з назвою вкладення.
  • path: TEXT. Містить шлях, за яким можна знайти вкладення.
  • fileType: TEXT. Містить розширенння вкладеного файлу.

Щоб прикріпити вкладення до завдання, воно має бути завантаженим на сервер.
